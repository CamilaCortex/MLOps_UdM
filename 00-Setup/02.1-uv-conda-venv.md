# Guía práctica: mismo proyecto, 3 entornos distintos

Vamos a crear un mini-proyecto de segmentación con KMeans (sklearn) y
ejecutarlo dentro de un notebook Jupyter. Lo haremos **3 veces**, una con cada
herramienta de entornos, para que los estudiantes vean las diferencias.

> **Resultado final:** 3 carpetas independientes en `~/Documents/`, cada una
> con su propio entorno, las mismas dependencias y el mismo notebook.

```
~/Documents/
├── example-knn-venv/
├── example-knn-uv/
└── example-knn-conda/
```

---

## 0. El notebook (común a los 3 proyectos)

Crea este archivo como `segmentation.ipynb` en cada proyecto. Puedes crearlo
manualmente en Jupyter o copiar el contenido de abajo.

### Celda 1 — Imports

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans
```

### Celda 2 — Generar datos sintéticos

```python
# 300 puntos, 4 grupos bien separados
X, y_true = make_blobs(
    n_samples=300,
    centers=4,
    cluster_std=0.60,
    random_state=42,
)

plt.scatter(X[:, 0], X[:, 1], s=30, alpha=0.7)
plt.title("Datos sin etiquetar")
plt.show()
```

### Celda 3 — Entrenar KMeans y visualizar

```python
kmeans = KMeans(n_clusters=4, random_state=42, n_init="auto")
kmeans.fit(X)
y_pred = kmeans.predict(X)

plt.scatter(X[:, 0], X[:, 1], c=y_pred, s=30, cmap="viridis", alpha=0.7)
plt.scatter(
    kmeans.cluster_centers_[:, 0],
    kmeans.cluster_centers_[:, 1],
    c="red", s=200, marker="X", edgecolors="black", linewidths=1.5,
    label="Centroides",
)
plt.title("Segmentación con KMeans (k=4)")
plt.legend()
plt.show()
```

### Celda 4 — Métricas rápidas

```python
from sklearn.metrics import silhouette_score

score = silhouette_score(X, y_pred)
print(f"Silhouette Score: {score:.3f}")
```

## 1. Proyecto con venv + pip

### Verificar que tienes pip

```bash
pip --version
# o
pip3 --version
# Esperado: pip 24.x.x from ... (python 3.x)

# Si no lo tienes, instálalo con:
# python3 -m ensurepip --upgrade
```

### Crear el proyecto

```bash
# ---------- Crear la carpeta y entrar ----------
mkdir -p ~/Documents/example-knn-venv
cd ~/Documents/example-knn-venv

# ---------- Crear entorno virtual ----------
python3 -m venv .venv

# Activar
# macOS/Linux:
source .venv/bin/activate
# Windows (PowerShell):
# .\.venv\Scripts\Activate.ps1

# ---------- Verificar ----------
python -V
which python    # debe apuntar a .venv/bin/python

# ---------- Instalar dependencias ----------
pip install -U pip
pip install numpy scikit-learn matplotlib jupyter

# ---------- Guardar dependencias ----------
pip freeze > requirements.txt

# ---------- Crear el notebook ----------
# Copia el contenido de la sección 0 en un archivo segmentation.ipynb
# o créalo desde Jupyter:
jupyter notebook
# Abre el navegador, crea un nuevo notebook y pega las celdas.

# ---------- Cuando termines ----------
deactivate
```

### Estructura final

```
example-knn-venv/
├── .venv/
├── requirements.txt
└── segmentation.ipynb
```

### Recrear el entorno (lo que haría un estudiante al clonar)

```bash
cd ~/Documents/example-knn-venv
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
jupyter notebook
```

### Nice to know: ¿por qué mi .venv tiene Python 3.14 y no 3.11?

`python3 -m venv .venv` usa **la versión de Python que esté primero en tu
PATH**. Si Homebrew instaló 3.14, esa es la que se usará. venv no te pregunta.

Si quieres forzar una versión específica (ej. 3.11):

```bash
# Opción 1: con pyenv
pyenv local 3.11.9
python -m venv .venv       # ahora usa 3.11.9

# Opción 2: con uv (más directo)
uv venv --python 3.11      # descarga 3.11 si no la tienes
```

> **Diferencia clave:** tanto uv como conda pueden instalar la versión de
> Python que necesites en un solo paso. Con venv, tú debes gestionar la
> versión de Python por separado (con pyenv u otra herramienta).

## 2. Proyecto con uv

### Verificar que tienes uv

```bash
uv --version
# Esperado: uv 0.x.x

# Si no lo tienes, instálalo:
# macOS/Linux: brew install uv
# Windows:     powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### Crear el proyecto

```bash
# ---------- Crear la carpeta y entrar ----------
mkdir -p ~/Documents/example-knn-uv
cd ~/Documents/example-knn-uv

# ---------- Inicializar proyecto con uv ----------
uv init

# ---------- Añadir dependencias ----------
uv add numpy scikit-learn matplotlib jupyter

# ---------- Verificar el entorno ----------
# uv ya creó .venv/ automáticamente al hacer uv add
source .venv/bin/activate
# Windows (PowerShell):
# .\.venv\Scripts\Activate.ps1

python -V
which python    # debe apuntar a .venv/bin/python

# ---------- Crear el notebook ----------
# Copia el contenido de la sección 0 en segmentation.ipynb
jupyter notebook

# ---------- Cuando termines ----------
deactivate
```

### Estructura final

```
example-knn-uv/
├── .venv/
├── .python-version
├── pyproject.toml      # dependencias declaradas aquí
├── uv.lock             # lockfile exacto (determinista)
├── hello.py            # archivo de ejemplo creado por uv init (puedes borrarlo)
└── segmentation.ipynb
```

### Recrear el entorno (lo que haría un estudiante al clonar)

```bash
cd ~/Documents/example-knn-uv
uv sync                 # lee pyproject.toml + uv.lock, instala todo
source .venv/bin/activate
jupyter notebook
```

> **Punto clave para la clase:** con uv no necesitas `requirements.txt`. El
> `pyproject.toml` declara las dependencias y `uv.lock` garantiza versiones
> exactas. Un solo comando (`uv sync`) recrea el entorno completo.

## 3. Proyecto con Miniconda

### Verificar que tienes conda

```bash
conda --version
# Esperado: conda 24.x.x

# Si no lo tienes, instálalo:
# macOS (Homebrew):  brew install --cask miniconda
# Windows:           winget install Anaconda.Miniconda3
# O descárgalo desde https://docs.anaconda.com/miniconda/install/
```

### Crear el proyecto

```bash
# ---------- Instalar Miniconda (si no lo tienes) ----------
# macOS (Homebrew):
brew install --cask miniconda
# macOS/Linux (instalador oficial):
# https://docs.anaconda.com/miniconda/install/

# Windows (PowerShell):
# winget install Anaconda.Miniconda3
# o descárgalo desde https://docs.anaconda.com/miniconda/install/

# Inicializar conda en tu shell (solo la primera vez)
conda init zsh    # o bash, powershell, etc.
# Reinicia la terminal después de esto.

# Aceptar términos de servicio (solo la primera vez)
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r

# ---------- Crear la carpeta ----------
mkdir -p ~/Documents/example-knn-conda
cd ~/Documents/example-knn-conda

# ---------- Crear entorno conda ----------
conda create -n knn-env python=3.11 -y

# ---------- Activar ----------
conda activate knn-env

# ---------- Verificar ----------
python -V
which python    # debe apuntar a .../envs/knn-env/bin/python

# ---------- Instalar dependencias ----------
conda install numpy scikit-learn matplotlib jupyter -y

# ---------- Exportar entorno ----------
conda env export > environment.yml

# ---------- Crear el notebook ----------
# Copia el contenido de la sección 0 en segmentation.ipynb
jupyter notebook

# ---------- Cuando termines ----------
conda deactivate
```

### Estructura final

```
example-knn-conda/
├── environment.yml
└── segmentation.ipynb
```

> Nota: el entorno de conda **no** vive dentro de la carpeta del proyecto.
> Está en `~/miniconda3/envs/knn-env/`. Esta es una diferencia importante
> respecto a venv y uv.

### Recrear el entorno (lo que haría un estudiante al clonar)

```bash
cd ~/Documents/example-knn-conda
conda env create -f environment.yml
conda activate knn-env
jupyter notebook
```

## 4. Resumen comparativo para la clase

| Aspecto | venv + pip | uv | Miniconda |
|---|---|---|---|
| **Crear entorno** | `python -m venv .venv` | `uv init` (automático) | `conda create -n env python=3.11` |
| **Activar** | `source .venv/bin/activate` | `source .venv/bin/activate` | `conda activate env` |
| **Instalar paquete** | `pip install numpy` | `uv add numpy` | `conda install numpy` |
| **Archivo de deps** | `requirements.txt` (manual) | `pyproject.toml` + `uv.lock` (auto) | `environment.yml` (manual) |
| **Recrear entorno** | `pip install -r requirements.txt` | `uv sync` | `conda env create -f environment.yml` |
| **Dónde vive el entorno** | `.venv/` en el proyecto | `.venv/` en el proyecto | `~/miniconda3/envs/` (global) |
| **Velocidad** | Lento | Muy rápido | Medio-lento |
| **Gestiona Python** | No (necesitas pyenv) | Sí (`uv python install`) | Sí (`python=3.11` al crear) |
